# Документация проекта

## 1. Общее описание

Веб-приложение предназначено для демонстрации учебных и практических работ.
Сайт используется как витрина выполненных заданий и проектов с упором на простую структуру и наглядный просмотр материалов.

Проект реализован как **SPA на React**. Данные загружаются асинхронно из **JSON-файлов**, расположенных в директории `public/data`. Этот подход позволяет изменять данные на сервере без необходимости пересборки приложения.

API, серверная логика, авторизация и SEO **не планируются**.
Все работы, практические страницы и страницы сайтов размещаются **на одном хостинге**.

## 5. Навигация и маршруты

### Основные маршруты

- `/practices` — Практические работы
- `/labs` — Лабораторные работы
- `/practical-pages` — Практические страницы
- `/sites` — Сайты
- `/courseworks` — Курсовые работы
- `/essays` — Рефераты

### Принципы навигации

- Клиентская маршрутизация (React Router DOM).
- **Детальные страницы для работ отсутствуют.**
- Все материалы открываются либо как файл (PDF / Word), либо как обычная страница сайта.

---

## 6. Источник данных

### Общие правила

- Все данные берутся **из JSON-файлов**, расположенных в директории `public/data`.
- Данные загружаются **асинхронно** с помощью `fetch` при монтировании компонентов.
- Такой подход позволяет обновлять JSON-файлы на сервере, и изменения сразу отразятся в приложении без необходимости его пересборки.
- API не планируется.

### Структура JSON-файлов

- **`works.json`**: Содержит практические и лабораторные работы, а также практические страницы.
  - `works`: `Work[]`
  - `practicalPages`: `PracticalPage[]`
- **`sites.json`**: Содержит информацию о сайтах. (`Site[]`)
- **`courseworks.json`**: Список курсовых работ. (`Coursework[]`)
- **`essays.json`**: Список рефератов. (`Essay[]`)

## 7. Типы данных (концептуально)

### 7.1 Работы (практические и лабораторные)

Тип `Work`:
```ts
{
  "number": number,
  "type": "practice" | "lab",
  "reportUrl": string
}
```

### 7.2 Практические страницы

Тип `PracticalPage`:
```ts
{
  "labNumber": number,
  "pages": [
    {
      "name": string,
      "url": string
    }
  ]
}
```

### 7.3 Сайты

Тип `Site`:
```ts
{
  "name": string, // Добавлено для идентификации
  "repositoryUrl": string,
  "figmaUrl": string,
  "pages": [
    {
      "name": string,
      "url": string,
      "references": string[] // Пути к изображениям
    }
  ]
}
```

### 7.4 Курсовые работы

Тип `Coursework`:
```ts
{
  "title": string, // Добавлено для отображения
  "pdfUrl": string
}
```

### 7.5 Рефераты

Тип `Essay`:
```ts
{
  "number": number,
  "pdfUrl": string
}
```

## 9. Архитектура проекта

### Подход

Используется модульная (feature-based) архитектура. Каждый раздел инкапсулирует UI, логику и асинхронную загрузку данных.

### Пример структуры проекта

```
public/
└─ data/
   ├─ works.json
   ├─ sites.json
   ├─ courseworks.json
   └─ essays.json
src/
├─ app/
│ ├─ router/
│ └─ layouts/
│
├─ features/
│ ├─ works/
│ ├─ practical-pages/
│ ├─ sites/
│ ├─ courseworks/
│ └─ essays/
│
├─ shared/
│ ├─ ui/
│ └─ types/
│
└─ main.tsx
```
